## 1. 硬件选型

### 1.1 主控

| 项目 | 规格 |
|------|------|
| 芯片 | ESP32-S3（推荐 WROOM-1 模组，内置 8MB PSRAM） |
| 主频 | 240MHz |
| Flash | 16MB（内置） |
| 通信 | WiFi 802.11 b/g/n / BLE 5.0 |
| 深度睡眠电流 | < 20µA（目标） |

### 1.2 显示屏

| 项目 | 规格 |
|------|------|
| 尺寸 | 4.2 inch |
| 色彩 | 三色（黑 / 白 / 红），驱动芯片 SSD1683 或等效 |
| 分辨率 | 400 × 300 px |
| 接口 | SPI |
| 刷新模式 | 全刷（三色同时刷新，~15s） / 局部刷新（仅黑白，~250ms） |
| 方向 | 支持 0° / 90° / 180° / 270° 旋转 |

> **三色刷新限制**：局部刷新仅支持黑白两色，红色层只能通过全刷更新（~15s）。因此正文阅读使用黑白局部刷新，红色仅用于 UI 强调元素（状态栏警告、选中边框、章节标题等）。红色元素的更新应合并到必要的全刷周期中，避免频繁触发三色全刷。

> **翻页策略**：阅读时使用黑白局部刷新（~250ms），每 5 次翻页执行一次黑白全刷清洁（~2s，防残影）。计数持久化到 NVS。三色全刷仅在红色元素需要更新时触发（如从书架进入阅读、警告状态变化等）。

### 1.3 存储

- MicroSD（SPI 接口，≥4GB），用于书籍文件与图片资源
- SPIFFS/LittleFS（内置 Flash），用于配置文件、字体缓存

### 1.4 电源

| 项目 | 规格 |
|------|------|
| 电池 | 3.7V 锂电池，JST-PH 2.0 接口 |
| 充电 | TP4056 或 MCP73831，支持 USB-C 充电 |
| 电量检测 | ADC + 分压电路，12bit 采样取平均 |
| 低电保护 | 电压 < 3.3V 时强制进入深度睡眠并提示 |

### 1.5 按键

共 4 个物理按键，均通过内部上拉 + 外部去抖（10ms 软件滤波）。

| 编号 | 位置建议 | 主功能 |
|------|----------|--------|
| BTN_PWR | 顶部 | 开关机 |
| BTN_PREV | 左侧上 | 上一页 / 向上 |
| BTN_NEXT | 右侧下 | 下一页 / 向下 |
| BTN_HOME | 底部中 | Home / 确认 |

### 1.6 RTC

推荐 DS3231（I²C），精度 ±2ppm，深度睡眠期间维持时间；亦可使用 SNTP 每次联网同步后写入 RTC。

## 2. 系统架构

### 2.1 固件分层

```
┌─────────────────────────────────────────────────────┐
│                  网页服务层 (esp_http_server)         │
│   书架管理 · WiFi配置 · 天气配置 · 休眠界面配置       │
├─────────────────────────────────────────────────────┤
│                  UI 页面状态机                        │
│  初始化→主页→书架→阅读→设置→休眠（单向/双向切换）    │
├───────────────┬─────────────────┬───────────────────┤
│   排版引擎    │   按键状态机     │   事件总线         │
│ FreeType渲染  │ 短按/长按/组合   │ (观察者模式)       │
├───────────────┴─────────────────┴───────────────────┤
│              硬件抽象层 (HAL)                         │
│  EPD驱动 · SD驱动 · ADC电量 · RTC · WiFi · OTA      │
└─────────────────────────────────────────────────────┘
```

### 2.2 核心数据结构（概要）

```cpp
// 系统配置（持久化到 NVS）
struct SysConfig {
    char     ssid[64];
    char     password[64];
    char     weatherApiKey[64];
    char     weatherCity[32];   // 和风天气 Location ID 或城市名
    uint8_t  sleepLayout;       // 0=时钟天气 1=风景图 2=诗词
    uint8_t  displayRotation;   // 0/1/2/3 → 0°/90°/180°/270°
    uint32_t sleepTimeoutSec;   // 无操作自动休眠秒数
    bool     initialized;       // 首次配网标志
};

// 阅读器设置（持久化到 NVS）
struct ReaderConfig {
    uint8_t  fontSize;          // 16/20/24/28/32px
    uint8_t  fontId;            // 字体索引
    uint8_t  lineSpacing;       // 1.0/1.2/1.5/2.0 (×10存储)
    uint8_t  charSpacing;       // 0–8px
    uint8_t  margin;            // 页边距 px
};

// 书目元数据（存储于 SD:/books/metadata.json）
struct BookMeta {
    char     id[16];
    char     title[128];
    char     author[64];
    char     filePath[256];
    uint32_t totalPages;
    uint32_t currentPage;
    uint32_t readingSeconds;
    uint8_t  coverOffset;       // 封面在文件中的字节偏移
};
```

## 3. 初始化流程

### 3.1 首次开机（`initialized == false`）

```
上电
  ↓
检测 NVS initialized 标志 → false
  ↓
进入 AP 模式
  SSID:  EReader-XXXX   (后四位为 MAC 末尾)
  密码:  ereader123      (可在编译时修改)
  IP:    192.168.4.1
  ↓
E-Ink 显示【配网引导页】
  ┌──────────────────────────┐
  │    欢迎使用 妙阅           │
  │                          │
  │  📶 连接 WiFi 进行初始化    │
  │                          │
  │  SSID:    EReader-A1B2   │
  │  密码:    ereader123     │
  │                          │
  └──────────────────────────┘
  ↓
用户在网页完成 WiFi 配置 → 重启
  ↓
进入正常启动流程
```

### 3.2 正常开机

```
上电 / 长按 BTN_PWR 唤醒
  ↓
HAL 初始化（SPI/I2C/ADC/SD）
  ↓
加载 NVS 配置
  ↓
连接已保存 WiFi（超时 10s，失败继续离线模式）
  ↓
SNTP 时间同步 + 写入 RTC
  ↓
加载书目索引（SD:/books/metadata.json）
  ↓
显示【主页】
```

## 4. 页面规格

### 4.1 全局状态栏（所有页面顶部）

高度：20px，全宽，深色反色背景（E-Ink 黑底白字）。

```
┌──────────────────────────────────────────────────┐
│  14:35                            🔋87%  📶      │
│  ←时间（左对齐）        电量+WiFi（右对齐，从右）      │
└──────────────────────────────────────────────────┘
```

- **时间**：`HH:MM`，从 RTC 读取，字号 14px
- **电量**：百分比 + 充电图标（充电中时动画刷新为静态闪烁替代方案），低于 20% 时电量数字显示红色（全刷时更新）
- **WiFi**：已连接显示信号图标；未连接显示 `✕`；AP 模式显示 `AP`
- 状态栏高度固定 20px，内容区从 y=20 开始

### 4.2 主页

内容区尺寸：400 × 280px（300-20 状态栏）

```
│                                   │
├──────────────────────────────────┤
│                                   │
│   ⛅ 北京                         │
│   晴转多云   26°C / 18°C          │ ← 天气区（居中）
│   空气质量:良  湿度:55%            │
```

- 天气数据来自和风天气 API，缓存 30 分钟，离线时显示上次缓存数据 + 时间戳

### 4.3 书架页

内容区：400 × 280px，网格布局。

- 默认每行 3 列，共 2 行，单屏显示 6 本；可翻页
- 每格 100 × 130px：上方封面图（从文件提取或默认占位图）+ 下方书名（2 行截断）
- 当前选中书格高亮（红色边框，全刷时更新）
- 底部显示翻页指示：`第 1 页 / 共 3 页`

### 4.4 阅读界面

```
┌──────────────────────────────────┐
│第三章 深渊中的黑暗森林   电量 + wifi │   ← 章节标题栏（20px）
├──────────────────────────────────┤
│                                  │ 
├──────────────────────────────────┤
│                                   │
│  正文内容区（排版引擎渲染）         │
│  支持 TXT / EPUB（纯文本提取）      │
│                                   │
│  支持自动分页，段落不跨页           │
│  孤行控制：段落最后一行不单独成页   │
```

- 正文区高度 = 300 - 20（状态栏）- 18（进度） = **262px**

分两个子菜单，通过 NEXT/PREV 在条目间移动，HOME 确认/进入。

|------|--------|------|
| 字体 | 思源宋体 / 思源黑体 / 霞鹜文楷 / 自定义 | 思源宋体 |
| 字号 | 16 / 18 / 20 / 22 / 24 / 28 / 32 px | 20px |
| 行间距 | 1.0 / 1.2 / 1.5 / 2.0 | 1.5 |
| 字间距 | 0 / 1 / 2 / 3 / 4 px | 1px |
| 页边距 | 4 / 8 / 12 / 16 px | 8px |

**系统设置**

| 条目 | 说明 |
|------|------|
| 显示方向 | 0° / 90° / 180° / 270°，立即生效并重绘 |
| 自动休眠 | 5 / 10 / 30 分钟 / 不自动 |
| 清洁屏幕 | 手动执行全刷清洁 |
| 恢复出厂 | 清除所有配置（需长按 HOME 3 秒确认） |
| 关于 | 妙阅  固件版本 · 编译日期 · MAC 地址 |

### 4.6 休眠界面

进入条件：
1. 用户短按 BTN_PWR
2. 无操作超过配置时长
3. 低电量触发（< 5%）

显示休眠界面 → 刷新 E-Ink → 关闭外设电源 → ESP32 深度睡眠

唤醒方式：BTN_PWR 短按或充电插入（可配置）。

## 5. 按键交互设计

### 5.1 按键事件定义

| 事件类型 | 触发条件 |
|----------|----------|
| 短按 | 按下并在 500ms 内释放 |
| 长按 | 按下超过 800ms 触发，持续触发（500ms 间隔重复） |
| 超长按 | 按下超过 2000ms 触发（仅 PWR 键使用） |
| 组合按 | 同时按下两个键（误触窗口 100ms） |

### 5.2 按键功能映射表

#### BTN_PWR（电源键）

| 场景 | 操作 | 行为 |
|------|------|------|
| 任何页面 | 超长按（2s） | 关机（显示关机画面 → 深度睡眠） |
| 休眠中 | 短按 | 唤醒，恢复到休眠前页面 |
| 任何页面 | 短按 | 进入休眠界面 |

#### BTN_PREV（上页键）

| 场景 | 操作 | 行为 |
|------|------|------|
| 阅读界面 | 短按 | 上一页 |
| 阅读界面 | 长按 | 快速向前翻 5 页 |
| 书架 | 短按 | 光标向上/向左移动 |
| 书架 | 长按 | 上一页（当书架有多页时） |
| 设置 | 短按 | 上移一个条目 |
| 设置（数值选项） | 长按 | 增加当前值 |
| 主页 | 短按 | 无操作 |

#### BTN_NEXT（下页键）

| 场景 | 操作 | 行为 |
|------|------|------|
| 阅读界面 | 短按 | 下一页 |
| 阅读界面 | 长按 | 快速向后翻 5 页 |
| 书架 | 短按 | 光标向下/向右移动 |
| 书架 | 长按 | 下一页（书架多页） |
| 设置 | 短按 | 下移一个条目 |
| 设置（数值选项） | 长按 | 减小当前值 |
| 主页 | 短按 | 进入书架 |

#### BTN_HOME（Home 键）

| 场景 | 操作 | 行为 |
|------|------|------|
| 任何页面（非主页） | 长按 | 返回主页 |
| 主页 | 短按 | 进入书架 |
| 书架 | 短按 | 打开选中书籍（进入阅读界面） |
| 阅读界面 | 短按 | 弹出上下文菜单（进度/跳转/退出） |
| 阅读界面（菜单中） | 短按 | 确认选中菜单项 |
| 设置 | 短按 | 确认当前选项 / 进入子菜单 |

#### 组合按键

| 组合 | 行为 |
|------|------|
| PREV + NEXT 同时长按 | 手动触发全刷清洁 |

### 5.3 阅读界面上下文菜单（HOME 短按弹出）

```
┌────────────────────────┐
```

## 6. 网页配置界面

访问地址：`http://<设备IP>/`（设备 IP 在主页或状态栏可查看）

技术栈：ESP-IDF `esp_http_server`，前端单页应用（HTML + CSS，无需框架），页面资源压缩后存入 SPIFFS。

### 6.1 书架管理

- 显示当前所有书籍列表（标题、作者、大小、最后阅读时间）
- **上传书籍**：支持 TXT / EPUB，最大单文件 50MB，上传后自动提取元数据
- **删除书籍**：勾选后确认删除（同时删除 SD 卡文件）
- **调整顺序**：拖拽排序，保存后同步到设备
- **上传封面**：为无封面书籍手动上传封面图（JPG，最大 200KB）

### 6.2 WiFi 配置

- 触发设备扫描附近 WiFi（返回 SSID 列表 + 信号强度）
- 点击 SSID → 弹出密码输入框 → 提交 → 设备尝试连接 → 返回结果
- 支持手动输入隐藏 SSID
- 显示当前连接状态、IP 地址、信号强度

### 6.3 天气配置

| 字段 | 说明 |
|------|------|
| API Key | 和风天气个人 API Key（明文存储，传输 HTTPS） |
| 城市/位置 | 支持城市名搜索（调用和风 GeoAPI）或直接输入 Location ID |
| 更新间隔 | 15 / 30 / 60 分钟（默认 30 分钟） |

配置保存后立即触发一次天气刷新。

### 6.4 休眠界面配置

提供三种布局（单选）：

**布局 A：时钟 + 天气**

```
┌──────────────────────────────────┐
│                                   │
│         14:35                     │
│      星期三 6月15日               │
│                                   │
│     ⛅ 北京  26°C                 │
│     晴转多云  湿度 55%            │
│                                   │
└──────────────────────────────────┘
```

**布局 B：风景图片**

- 网页上传自定义图片（JPG，建议 400×300 像素），自动转换为三色 BMP（黑/白/红抖动算法）
- 图片存于 `SD:/sleep/custom.bmp`
- 若未上传，显示内置默认图（山水线稿）
- 可选：在图片左下角叠加时间（小字，半透明蒙层）

**布局 C：诗词 + 时间**

- 随机显示内置诗词库（100 首，存于 SPIFFS `poems.json`）
- 布局：
  ```
  ┌──────────────────────────────────┐
  │                                   │
  │      君不见，黄河之水天上来，      │
  │      奔流到海不复回。             │
  │                                   │
  │                  ——李白《将进酒》  │
  ```

## 7. 电源管理

### 7.1 运行模式

| 模式 | 描述 | 预计电流 |
|------|------|----------|
| 活跃 | CPU 运行，WiFi 活跃，屏幕保持画面 | ~100–150mA |
| WiFi 关闭 | 仅阅读，关闭 WiFi radio | ~30–50mA |
| 轻度睡眠 | 保持 RAM，等待按键唤醒 | ~2–5mA |
| 深度睡眠 | 只有 RTC 运行，仅 BTN_PWR 唤醒 | < 0.1mA |

### 7.2 自动休眠逻辑

```
无操作计时器启动
  ↓ 超过 [设置的超时时间]
刷新休眠界面（E-Ink 最后一次刷新）
  ↓
关闭 WiFi、SD 卡、外设电源
  ↓
保存当前阅读位置到 NVS
  ↓
进入 ESP32 Deep Sleep
  ↓
[BTN_PWR 中断唤醒]
  ↓
正常开机流程
```

### 7.3 电量显示

- 使用 ADC 对分压后的电池电压采样（建议 16 次平均）
- 电压 → SOC 映射表（锂电池放电曲线，非线性）
- 充电时显示充电动画（静态 E-Ink 方案：每 5 分钟刷新一次进度格）
- 低于 15% 时状态栏图标显示红色警告
- 低于 5% 时强制保存并进入深度睡眠，显示"电量不足，请充电"

## 8. 存储规格

### 8.1 SD 卡目录结构

```
SD:/
```

## 9. 字体与排版引擎

### 9.1 字体分类

| 用途 | 字体方案 |
|------|----------|
| UI 标签、状态栏 | 预生成点阵字体（14px），存于 SPIFFS，快速渲染 |
| 正文 | TTF 矢量字体，运行时通过 FreeType 渲染到位图缓冲区 |

### 9.2 分页算法

```
输入：原始文本、字号、字间距、行间距、页面宽高、页边距

1. 按段落切分文本
2. 对每段：按字符逐一测量宽度，遇到换行触发点（标点、空格、CJK字符边界）决定断行
3. 累积行高，当超过页面可用高度时，当前段落整体移至下一页（避免段落首行孤行）
4. 生成 PageIndex 数组：[{startByte, startLine, pageNum}]
5. 翻页时按 PageIndex 跳转，无需重新排版
```

### 9.3 支持的文件格式

| 格式 | 支持情况 |
|------|----------|
| TXT | 完整支持，自动检测 UTF-8 / GBK 编码 |
| EPUB | 支持纯文本提取（解析 content.opf + HTML 去标签），不支持复杂 CSS 排版 |
| PDF | **不支持**（ESP32 资源不足） |
| MOBI / AZW | **不支持** |

## 10. 天气服务集成

使用**和风天气**（QWeather）免费版 API。

### 10.1 API 调用

| 接口 | 用途 | 调用频率 |
|------|------|----------|
| `https://geoapi.qweather.com/v2/city/lookup` | 城市搜索 | 配置时调用 |
| `https://devapi.qweather.com/v7/weather/now` | 实时天气 | 每 30 分钟 |
| `https://devapi.qweather.com/v7/weather/3d` | 3 天预报 | 每 2 小时 |
| `https://devapi.qweather.com/v7/air/now` | 空气质量 | 每 30 分钟 |

### 10.2 缓存策略

- 数据缓存到 NVS（JSON 序列化），带时间戳
- 离线或 API 失败时显示缓存数据 + "⚠ 数据来自 XX 分钟前"
- 缓存过期时间：天气 60 分钟，预报 4 小时

## 11. 休眠界面规格

### 11.1 进入/退出

| 事件 | 行为 |
|------|------|
| 进入 | 刷新 E-Ink → 最小化功耗（见电源管理）→ Deep Sleep |
| 退出 | BTN_PWR 短按 → 正常启动 → 恢复到休眠前页面 |

### 11.2 布局刷新

休眠界面进入深度睡眠前完成 E-Ink 刷新，睡眠期间屏幕内容不变（E-Ink 掉电保持）。

如需定时刷新休眠界面（如时钟），使用 ESP32 的 ULP 协处理器定时唤醒（不推荐，耗电增加），或在配置中禁用此功能，仅显示静态画面。

## 12. 非功能性需求

| 指标 | 目标值 |
|------|--------|
| 翻页延迟（局部刷新） | ≤ 300ms |
| 全刷延迟（黑白） | ≤ 2s |
| 全刷延迟（三色） | ≤ 15s |
| 冷启动到主页显示 | ≤ 5s（WiFi 连接异步进行） |
| 待机续航（深度睡眠） | ≥ 4 周（1800mAh） |
| 连续阅读（WiFi 关闭） | ≥ 12 小时（1800mAh） |
| 书架容量 | ≤ SD 卡空间，书目索引支持 ≥ 500 本 |
| OTA 升级 | 支持通过网页上传固件（`/update` 接口） |
| 崩溃恢复 | 看门狗超时自动重启，重启后恢复阅读位置 |

## 13. 开发阶段划分

### Phase 1：基础框架（约 3 周）

- PlatformIO 项目初始化（`platformio.ini` 配置 `framework = esp-idf`），组件管理
- 依赖组件确认：EPD 驱动（SPI GPIO 手写驱动或移植 GxEPD2）、FreeType（ESP-IDF 组件）、cJSON（内置）
- E-Ink 驱动验证：三色全刷、黑白局部刷新、红色层更新时序
- HAL 层：按键去抖、ADC 电量、RTC 读写
- NVS 配置读写
- 页面状态机骨架（空白页面切换）

### Phase 2：核心 UI（约 4 周）

- 字体渲染（点阵 UI 字体 + FreeType 正文字体）
- 状态栏组件
- 主页（日期 + 静态占位天气 + 统计）
- 书架页（SD 扫描 + 封面渲染 + 网格布局）
- 阅读界面（TXT 解析 + 分页算法 + 翻页）
- 设置页（菜单导航 + 配置读写）

### Phase 3：网络与服务（约 3 周）

- WiFi 连接管理（STA/AP 切换，`esp_wifi` API）
- SNTP 时间同步（`esp_sntp`）
- 和风天气 API 集成（`esp_http_client`）
- `esp_http_server` 搭建
- 网页配置界面（书架管理 + WiFi + 天气 + 休眠配置）

### Phase 4：完善与优化（约 2 周）

- 休眠界面三种布局
- 电源管理（深度睡眠 / 自动休眠）
- EPUB 支持
- OTA 升级
- 按键全场景测试
- 续航测试与优化
